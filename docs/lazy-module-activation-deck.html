How It Works

Agent configuration declares tools

# In agent definition (e.g., shadow-operator.md) --- meta: name: shadow-operator description: Manages shadow environments tools: activate: - shadow # ‚Üê Lazy activated on spawn --- # When this agent spawns: # 1. Amplifier sees the tool declaration # 2. Resolves the module (async, with fallback) # 3. Activates it in the agent's session only # 4. Parent session unchanged
